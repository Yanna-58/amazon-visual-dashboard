## 📌 使用前必读：我要导什么报告？

| 你想做什么 | 需要的报告类型 | 看板里上传到哪 |
|-----------|--------------|-------------|
| 看新品的整体表现 | Business Report **By ASIN** | 导入 BUSINESS REPORTS 数据 |

## 🆕 场景一：新品分析

### 适用情况
- 产品刚上架不久（1-3个月内）
- 想判断：主图/A+/橱窗图做得怎么样？
- 想发现：哪些新品有潜力？哪些需要优化？

### 📊 领星ERP 操作步骤

```
第1步：打开领星 → 数据中心 → 报告导出

第2步：点击「添加报告」，填写：
├── 目标店铺：选择对应站点（如美国站）
├── 时间范围：从上架日期开始 → 到今天
│   （建议至少选最近30天，太少没意义）
├── 报告类型：Business Report
└── 详细类型：⭐ By ASIN（按产品明细）← 关键！

第3步：提交后等待「已生成」→ 点【下载】

第4步：下载的CSV文件 → 上传到看板
```

### 📊 亚马逊后台操作步骤（备选）

```
第1步：登录 sellercentral.amazon.com

第2步：左侧菜单 → 数据报告 (Business Reports)

第3步：左侧子菜单选择：
   └─ By ASIN: 按产品明细 ← 选这个！不要选By Date

第4步：页面顶部设置：
   ├── 日期范围：从上架日期开始 → 到今天
   └── 点击右上角「下载」(Download) 图标

第5步：得到的文件 → 上传到看板
```

## 👑 场景二：老品日常监控

### 适用情况
- 产品已上线很久（几个月~几年）
- 定期查看表现、发现异常下降的产品
- 判断是否需要更新主图/A+

### 📊 领星ERP 操作步骤

```
第1步：同上 → 报告导出

第2步：「添加报告」，填写：
├── 目标店铺：老品所在站点
├── 时间范围：
│   ├─ 日常监控：最近30天
│   └─ 看长期趋势：最近90天 或 最近180天 ← 推荐！
├── 报告类型：Business Report
└── 详细类型：⭐ By ASIN

第3步：等待生成 → 下载 → 上传看板

💡 建议：设为定时任务，每周自动导出一次
```

### 💡 老品 vs 新品的区别

| 项目 | 新品 | 老品 |
|------|------|------|
| **时间范围** | 30天够用 | 建议 90-180 天（看出长期趋势） |
| **关注指标** | CTR 是否及格（有没有人点进来） | CVR 是否稳定（点了之后买不买） |
| **常见问题** | 曝光不够 / CTR 太低 | CTR/CVR突然下降 / 竞品超越 |
| **优化方向** | 主图吸引力、标题关键词 | A+内容、价格竞争力、评价积累 |

## 🔥 场景三：优化前后对比（最实用！）

### 适用情况
- 老品刚换了新主图 / 新A+ / 新橱窗图
- **想知道：改完之后效果好不好？**
- 用数据证明设计的价值（"你看！换了我做的图，CTR涨了15%！"）

### ⚠️ 这是最关键的：你需要两份数据！

```
优化前数据 + 优化后数据 = 对比效果
     ↓              ↓
  换图之前         换图之后
  的30天数据       的30天数据
```

### 📊 完整操作步骤

#### 第1步：记录优化的时间节点

```
在日历/备忘录记下：

产品：B0XXXXX1（女式运动裤）
优化内容：更换主图 + 重做 A+ 页面
优化日期：2026-04-01（假设这天上线新图）

→ 优化前的数据 = 2026-03-01 到 2026-03-31（优化前30天）
→ 优化后的数据 = 2026-04-01 到 2026-04-30（优化后30天）
                    ↑ 至少等14-30天再导！数据才有统计意义
```

#### 第2步：导出优化前的数据（历史数据）

```
方法A：找运营要归档的历史报告
   "能不能帮我找一下 3月份的 Business Report By ASIN？"
   
方法B：在亚马逊后台自己拉历史数据
   业务报告 → By ASIN → 时间选 3月份 → 下载
   
方法C：领星历史任务
   报告导出 → 看之前的任务记录 → 如果还在90天内就能重新下载
```

> ⚠️ **重要：亚马逊历史报告只保留有限时间！**
> - 后台一般保留 **最近90天** 的报告可下载
> - 领星的报告记录只保留 **14天**
> - **所以：优化前一定要先存一份数据！或者让运营定期归档**

#### 第3步：等优化生效后再导优化后数据

```
最少等：14天（太短不靠谱）
推荐等：30天（数据更准确）
理想等：一个完整自然月

为什么不能马上导？
→ 亚马逊算法需要时间收录新图片
→ 数据量太少会有偶然性（某天刚好有个大单就虚高了）
```

#### 第4步：上传到看板对比

```
打开看板 → 「视觉优化前后对比」标签页

🔴 优化前 (Before) 区域：
   → 上传优化前的 Business Report CSV
   → 可选：上传优化前的广告报表

🟢 优化后 (After) 区域：
   → 上传优化后的 Business Report CSV  
   → 可选：上传优化后的广告报表

→ 点【开始对比】→ 看到涨跌百分比 + 视觉归因建议
```

## ❓ 常见问题 FAQ

### 关于数据本身

**Q1：导出来的数据是空的 / 只有几行？**
- 可能原因：选了错误的日期范围或站点
- 可能原因：该产品在该时间段确实没有流量
- 解决：扩大日期范围，确认产品和站点正确

**Q2：找不到某个ASIN的数据？**
- 最常见：导出的是 By Date 而不是 By ASIN → By Date 里没有ASIN列
- 产品太新且无流量 → 不会出现在报告中
- 产品已下架/停售 → 不再有数据
- 解决：确保选 **By ASIN 类型**

**Q3：领星显示"待执行"一直不变？**
- 亚马逊API限制每小时约6次请求
- 任务多时会排队，可能等几个小时
- 第二天再看应该就好了
- 或者联系公司IT/运维检查领星配置

**Q4：报告超过90天还能下载吗？**
- 亚马逊后台：通常不行（超时失效）
- 领星：任务记录14天后清除
- **结论：重要数据及时保存！**

### 关于权限

**Q5：我没有亚马逊后台账号怎么办？**
- 找运营要数据（最简单）
- 申请一个只读权限的子账号（最自主）
- 让运营在领星设置定时导出发给你（最省事）

**Q6：运营说"太忙了没空导"？**
- 这个操作真的只需要30秒
- 可以让运营设置领星定时任务，自动导出
- 或者跟老板说一下，这是为了提升团队效率

### 关于使用看板

**Q7：一次可以上传多个产品的数据吗？**
- 可以！By ASIN 报告包含所有产品
- 看板会自动分组，每个ASIN独立展示
- 你可以在产品总览页筛选你想看的

**Q8：广告数据必须上传吗？**
- 不是必须的！业务数据已经足够分析视觉健康度
- 广告数据是锦上添花，帮助看ACOS和点击花费
- 没有广告数据也能正常使用所有核心功能

**Q9：数据多久更新一次合适？**
- 日常监控：**每周或每两周**更新一次
- 新品追踪：**每周**关注（新品变化快）
- 老品对比：**每月或每季度**看一次大趋势
- 优化验证：**优化后等30天**再对比

## 📋 快速检查清单（每次导出前对照）

```
□ 确认站点正确（美站/欧站/日站？）
□ 确认时间范围合理（太短没意义，太长不够精准）
□ 确认类型是 By ASIN（不是 By Date！）
□ 文件格式是 CSV 或 TXT
□ 文件大小不超过 10MB（太大就在Excel里精简）
□ 优化对比场景：确认有"优化前"+"优化后"两份数据
```
