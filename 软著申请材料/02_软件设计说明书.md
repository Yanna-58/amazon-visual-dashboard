## 一、软件开发背景与目的

### 1.1 背景

在跨境电商行业中，视觉设计师负责亚马逊平台的产品主图（Listing Image）、副图、A+页面等视觉素材的设计工作。传统工作流程中，设计师**无法直接获取产品的业务数据表现**，导致以下问题：

- 设计决策依赖主观判断，缺乏数据支撑
- 无法量化评估视觉优化对 CTR（点击率）、CVR（转化率）的实际影响
- 与运营团队沟通时缺乏共同的数据语言
- 多产品并行时难以系统化地跟踪每个产品的视觉健康度

### 1.2 目的

本软件旨在为**跨境视觉设计师**打造一款**零门槛、开箱即用**的数据分析工具，实现：

1. **数据驱动设计决策** — 通过可视化图表直观呈现 CTR/CVR 等核心指标
2. **多维度分析能力** — 支持按日期趋势、按 ASIN 产品维度、环比同比对比、优化前后对比
3. **团队协作支持** — 设计师标签分配、多产品并排追踪
4. **完全本地运行** — 无需注册、无需后端、保护商业数据安全

## 三、技术架构

### 3.1 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| **前端框架** | 原生 HTML5 + CSS3 + JavaScript ES6+ | 单文件应用，无构建工具依赖 |
| **图表库** | Chart.js v4.4.7（CDN引入） | 用于所有趋势图、柱状图、双轴图的渲染 |
| **数据存储** | localStorage API | 浏览器本地持久化，无需后端服务器 |
| **数据处理** | 自研CSV解析器（含模糊匹配、多编码支持） | 核心算法模块 |
| **样式方案** | CSS Custom Properties（CSS变量）+ Flex/Grid布局 | 支持暗色模式的主题切换 |
| **响应式设计** | CSS Media Query | 适配桌面端和平板端 |
| **部署方式** | 静态文件（HTML单文件） | 可本地打开也可通过任意静态托管服务发布 |

### 3.3 核心算法描述

#### 3.3.1 字段模糊匹配算法

```
输入：CSV原始表头数组、候选关键字列表
输出：匹配到的字段值

算法逻辑：
1. 将表头统一转小写
2. 遍历候选列表中的每个关键词
3. 对每个表头检查是否包含该关键词（includes方法）
4. 返回第一个匹配到的值
5. 若无匹配则返回 null

示例：
  候选: ['Sessions', '会话']
  匹配: "Daily Sessions" ✅ → 返回该列的值
  匹配: "会话数" ✅ → 中文也能匹配
  匹配: "session_count" ✅ → 包含 session 即可
```

#### 3.3.2 视觉健康度评估算法

```
输入：某产品的 CVR（转化率）、PV/Session（人均页面浏览量）

评分规则：
  if (CVR >= 12% && PV/Session >= 2.5) → 🟢 good（优秀）
  else if (CVR >= 8% || (CVR >= 6% && PV/Session >= 2.5)) → 🟡 warn（中等）
  else → 🔴 bad（需优化）

设计依据：
  - CVR ≥ 12% 为亚马逊服装类目优秀水平
  - PV/Session ≥ 2.5 表示用户愿意深入浏览详情页
  - 两者结合反映"主图吸引力 + 详情页说服力"的综合效果
```

## 五、软件特色与创新点

1. **面向视觉设计师的专业工具** — 目前市场上尚无专门针对跨境电商设计师的数据分析工具，填补了细分领域空白
2. **零技术门槛** — 无需安装、无需数据库、无需API配置，一个 HTML 文件即完整可用
3. **模糊匹配技术** — 自研的中英文字段匹配算法，兼容亚马逊不同站点、不同版本的导出格式
4. **视觉健康度评估体系** — 将抽象的 CTR/CVR 数值转化为设计师易懂的 🟢🟡🔴 健康度评级
5. **完全本地数据处理** — 所有数据在浏览器本地完成计算和存储，不经过任何第三方服务器，保障商业数据安全
6. **MIT 开源协议** — 以开源方式发布，促进社区协作和知识共享
