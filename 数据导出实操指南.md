## 🤔 先搞懂一件事：数据从哪来？

你的看板需要的数据来源有 **3 条路**：

```
路径A：亚马逊后台（原始）→ 你自己导出
路径B：领星ERP（中间层）→ 通过领星批量请求
路径C：找运营要 → 最省事但依赖别人
```

**推荐顺序：** 先试路径B（领星），不行再走路径C（找运营）。

---

## 🔑 核心概念：两种报告类型

这是最关键的一点！亚马逊业务报告有两种模式：

| 模式 | 按日期 (By Date) | 按ASIN (By ASIN) |
|------|:----------------:|:----------------:|
| **每一行代表什么？** | 一天 | 一个产品(ASIN) |
| **适合看什么？** | 时间趋势、整体店铺表现 | **每个产品单独分析** ⭐ |
| **设计师用哪个？** | 辅助参考 | **这个是主力！** ✅ |
| **能筛选产品吗？** | 不能，全店汇总 | 可以选特定ASIN |

### 打个比方：
```
按日期报告 = 公司的每日营收表（每天赚多少钱）
按ASIN报告 = 每个产品的成绩单（哪个产品好/不好）

→ 设计师要看的是每个产品的成绩单！
```

## 📋 路径B：领星ERP 导出步骤（推荐）

### 第一步：进入报告导出页面

```
领星ERP → 数据 → 报告导出（或：数据中心 → 报告导出）
```

> 💡 如果你找不到入口，直接问运营："领星的报告导出功能在哪里？"

### 第二步：添加报告任务

点击「添加报告」，填写：

| 配置项 | 填写内容 |
|--------|---------|
| **目标店铺** | 选择你要分析的站点（美国站/欧洲站/日本站等） |
| **请求时间范围** | 建议：最近 **30天** 或 **90天** |
| **报告类型** | 选 **Business Report（业务报告）** |
| **详细类型** | 选 **By ASIN** ← 关键！不要选 By Date |

### 第三步：等待生成

提交后状态变化：
```
待执行 → 生成中（可能几分钟到几小时）→ 已生成
                                    ↓
                              点【下载】获取CSV文件
```

> ⚠️ 注意：亚马逊API有限制，每小时约6次请求。如果任务多可能排队。

### 第四步：下载并上传看板

下载得到的 CSV 文件 → 直接上传到看板的「导入 BUSINESS REPORTS 数据」区域。

## 🎯 关于你提到的具体问题

### Q1：5个新品上架了1-3个月，要分别下载吗？

**不用分别下载！一次搞定：**

| 方法 | 操作 |
|------|------|
| **方法1（推荐）** | 导出 **By ASIN** 报告 → 包含所有产品 → 上传后看板自动按ASIN分组展示 |
| **方法2** | 如果只想看5个新品 → 导出后在Excel里删掉其他行 → 再上传看板 |

**老品也是一样**——一次导出所有产品，上传后看板自动分组，你只关注你需要的产品就行。

### Q2：用ASIN查找却找不到该产品？

这很常见！可能的原因：

| 可能原因 | 解决办法 |
|----------|---------|
| **① 产品太新，还没产生流量** | 上架不到24小时或没有任何Session的ASIN不会出现在报告中 → 等几天再导 |
| **② 导出的是"按日期"不是"按ASIN"** | 按日期报告中没有ASIN列 → 确保选了 By ASIN 类型 |
| **③ 时间范围不对** | 产品下架期间不会有数据 → 扩大时间范围试试 |
| **④ 该ASIN在另一个站点** | 美国站和日本站是分开的报告 → 确认站点是否正确 |
| **⑤ 领星数据同步延迟** | 领星可能还没同步最新数据 → 等24小时再试 |

### Q3：要不要直接找运营要数据？

**可以！而且不会增加他们工作量。** 教你一个聪明的做法：

## 💬 路径C：找运营要数据（话术模板）

### 发给运营的消息（直接复制）

> 嗨~ 我最近做了一个数据分析小工具，想看看我们产品的视觉表现数据。
>
> 能麻烦帮我导出一下 **Business Report（按ASIN明细）** 吗？
> 
> 具体要求：
> - 站点：美国站（或其他你需要的）
> - 时间范围：最近30天
> - 类型：**By ASIN（按产品）**，不是 By Date 哦
> - 格式：CSV 就行
>
> 如果领星能批量导出的话最好，或者亚马逊后台的也行~
>
> 谢谢啦！🙏

### 为什么这不增加工作量？

| 运营要做的事 | 工作量 |
|-------------|--------|
| 在领星点几下导出 | **30秒** |
| 在亚马逊后台下载CSV | **1分钟** |
| 他们本来就要定期看的 | **零额外工作** |

大部分运营每周/每月都会自己导出这些数据，你只是让他们顺手发一份给你而已。

## 📥 看板支持的两类数据总结

### ① 业务数据（必须有的）

```
来源：亚马逊后台 → 数据报告 → 业务报告
      或：领星ERP → 报告导出

你需要：By ASIN 类型的 CSV 文件
关键字段：Date / Sessions / Page Views / Unit Session % / Ordered Product Sales / ASIN
用途：产品总览、健康度色标、趋势变化、新品追踪、环比、同比
```

### ② 广告数据（可选的）

```
来源：亚马逊后台 → 广告 → 广告报表 → 搜索词/投放
      或：领星ERP → 广告报表导出

你需要：商品推广/品牌推广的搜索词/投放报表 CSV
关键字段：Campaign / Impressions / Clicks / CTR / CPC / Spend / Sales
用途：广告花费分析、ACOS判断、优化对比时的广告维度
```

> 💡 **如果暂时没有广告数据也没关系！** 业务数据已经够用了，广告数据是锦上添花。

## ❓ 常见问题 FAQ

**Q: 必须要用领星吗？**
A: 不是！亚马逊后台也能导出。后台路径：数据报告 → 业务报告 → 右上角下载CSV。

**Q: 导出的文件太大怎么办？**
A: 看板限制10MB。如果太大，在Excel里只保留需要的列和行就行。

**Q: 多个站点的数据怎么处理？**
A: 每个站点单独导出一份，看板支持切换站点（美/欧/日）。

**Q: 数据多久更新一次合适？**
A: 建议每周或每两周。太频繁意义不大，亚马逊数据本身有延迟。

**Q: 运营不愿意给怎么办？**
A: 可以自己在亚马逊后台导出——只要你有卖家账号权限就能下载。
