## 🔑 核心要点提醒（发给运营前自己确认）

### ✅ 必须要强调的事

| 要点 | 为什么重要 |
|------|-----------|
| **必须是 By ASIN，不是 By Date** | By Date 是每天汇总一行，没有产品维度；By ASIN 才能看每个产品的表现 |
| **时间范围从 2月1日开始** | 你最早的品 2月16日上线，至少要覆盖到它 |
| **不需要筛选产品** | 让运营全店导出最省事，你在看板里自己筛选需要的SKU |
| **格式随便** | CSV / TXT 都支持，怎么方便怎么来 |

### ⚠️ 常见坑

| 坑 | 后果 | 怎么避免 |
|----|------|---------|
| 运营导了 By Date | 文件里没有ASIN列，无法按产品分组 | 话术里反复强调 **By ASIN** |
| 时间范围太短 | 新品数据不全，分析没意义 | 明确写 **2026-02-01 至今** |
| 只导了部分产品 | 运营反而要多操作筛选 | 强调 **全店导出，不用筛选** |

## 📊 运营导出的文件长什么样？

> 给你一个预期，收到文件后核对一下表头

### ✅ 正确的表头应该包含这些关键字段：

```
Date, Parent ASIN, ASIN, Product Title, Sessions, Page Views, 
Buy Box Percentage, Units Ordered, Unit Session Percentage, 
Ordered Product Sales, Total Order Items, Average Selling Price
```

### 收到文件后检查清单：

- [ ] 有 `Date` 列吗？（日期）
- [ ] 有 `ASIN` 或 `Parent ASIN` 列吗？（产品编码）
- [ ] 有 `Sessions` 列吗？（会话数/流量）
- [ ] 有 `Unit Session Percentage` 列吗？（转化率）
- [ ] 有 `Ordered Product Sales` 列吗？（销售额）
- [ ] 行数 > 0 且不是只有一行表头？

> **只要上面这几列都有，基本就没问题！** 其他列有没有不影响看板使用。

## ❓ 可能会遇到的问题 & 应对

### Q1: 运营说"领星导不了这么久的"
- 亚马逊 API 一般保留 **90天** 历史数据可拉取
- 你的新品最早 2月16日，到现在还在90天内，没问题
- 如果真超了就让他尽量拉最长的，或者分段导（2-3月一段 + 4-5月一段）

### Q2: 导出来的文件很大（几MB）
- 看板限制 **10MB** 以内
- 如果太大，让运营在Excel里只保留必要的列就行
- 或者你在 Excel 里删掉其他老品的数据行，只留这10个SKU相关的

### Q3: 找不到某个父SKU的数据
- Business Report 显示的是 **ASIN** 不是父SKU
- 一个父SKU下面可能有多个子ASIN（不同尺码/颜色）
- 这是正常的，看板会自动把同父ASIN的子体汇总

### Q4: 有些新品数据很少/几乎为零
- 可能原因：上架时间短、没开广告、没流量
- 不影响导入，看板会显示实际数据，你就能看到"这个品曝光严重不足"
